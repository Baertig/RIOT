    1 # Build In Docker {#build-in-docker}
    2 
    3 Some parts of RIOT's build process can be performed inside a Docker container,
    4 which comes with the necessary compilers and toolchains and is fully managed by
    5 the build system. It can be enabled by passing `BUILD_IN_DOCKER=1` to make.
    6 
    7 ```shell
    8 $ BUILD_IN_DOCKER=1 make
    9 ```
    10 
    11 If your user does not have permissions to access the Docker daemon:
    12 
    13 ```shell
    14 $ BUILD_IN_DOCKER=1 DOCKER="sudo docker" make
    15 ```
    16 
    17 to always use Docker for building, set `BUILD_IN_DOCKER=1` (and if necessary
    18 `DOCKER="sudo docker"`) in the environment:
    19 
    20 ```console
    21 $ export BUILD_IN_DOCKER=1
    22 ```
    23 
    24 ## Targets ran in Docker: DOCKER_MAKECMDGOALS_POSSIBLE
    25 
    26 Currently only build related targets are ran in the docker container, the exact
    27 list is under `DOCKER_MAKECMDGOALS_POSSIBLE` variable.
    28 
    29 ## Environment Variables: DOCKER_ENV_VARS
    30 
    31 When building in docker one might want for the command ran in docker to inherit
    32 variables that might have been set in the command line. e.g.:
    33 
    34 ```shell
    35 BOARD=samr21-xpro USEMODULE=xtimer make -C examples/basic/hello-world
    36 ```
    37 
    38 In `docker.ink.mk` the origin of a variable listed in `DOCKER_ENV_VARS` is checked
    39 and if the origin is `environment` or `command` (for make command-line argument)
    40 then those variables will be passed to the docker container.
    41 
    42 You can also set in `DOCKER_ENV_VARS` in the environment to add variables to the
    43 list, e.g.:
    44 
    45 ```shell
    46 DOCKER_ENV_VARS=BEER_TYPE BEER_TYPE="imperial stout" BUILD_IN_DOCKER=1 make -C examples/basic/hello-world/
    47 docker run --rm -t -u "$(id -u)" \
    48  ...
    49  -e 'BEER_TYPE=imperial stout' \
    50  -w '/data/riotbuild/riotbase/examples/basic/hello-world/' \
    51  'riot/riotbuild:latest' make
    52 ```
    53 
    54 Your application Makefile can also extend `DOCKER_ENV_VARS`.
    55 
    56 ### Directly Define Environment Variables: DOCKER_ENVIRONMENT_CMDLINE
    57 
    58 `DOCKER_ENVIRONMENT_CMDLINE` can be used to add variables directly to the environment
    59 but will need to be prefixed with `-e` (see [option-summary]).
    60 
    61 e.g.:
    62 
    63 ```
    64 DOCKER_ENVIRONMENT_CMDLINE='-e BEER_TYPE="imperial stout"' BUILD_IN_DOCKER=1 make -C examples/basic/hello-world/
    65 docker run --rm -t -u "$(id -u)" \
    66  ...
    67  -e 'BEER_TYPE=imperial stout' \
    68  -w '/data/riotbuild/riotbase/examples/basic/hello-world/' \
    69  'riot/riotbuild:latest' make
    70 ```
    71 
    72 ## Command Line Variables: DOCKER_OVERRIDE_CMDLINE
    73 
    74 Command line arguments are variables that are passed after `make` e.g.
    75 `make target FOO=bar`, but different to environment variables a variable defined
    76 through the command line will take precedence over all assignments of `FOO` within
    77 the makefile (same effect as adding `-e` for environment variables, see
    78 [option-summary] for more details.
    79 
    80 To pass variables overriding the command-line to docker `DOCKER_OVERRIDE_CMDLINE`
    81 may be used:
    82 
    83 ```shell
    84 DOCKER_OVERRIDE_CMDLINE="BEER_TYPE='imperial stout'" BUILD_IN_DOCKER=1 make -C examples/basic/hello-world/ RIOT_CI_BUILD=1
    85 Launching build container using image "riot/riotbuild:latest".
    86 sudo docker run --rm -t -u "$(id -u)" \
    87  ...
    88  -w '/data/riotbuild/riotbase/examples/basic/hello-world/' \
    89  'riot/riotbuild:latest' make BEER_TYPE='imperial stout' 'RIOT_CI_BUILD=1'
    90 ```
    91 
    92 ### Redefined or Overridden Variables: DOCKER_ENV_VARS_ALWAYS
    93 
    94 There is a corner case for the handling of `DOCKER_ENV_VARS`. If a variable is
    95 redefined (`+=`, `=`, `:=`) or overridden then the origin of the variable will be changed
    96 to `file` and there is no way of detecting in Make how it was set.
    97 
    98 If this happens after `docker.ink.mk` this is not an issue, but for all variables
    99 susceptible to be defined in the application `Makefile` this is indeed the case.
    100 
    101 A subset of these variables, namely variables relating to dependency resolution
    102 are therefore unconditionally passed to docker. The complete list can be found
    103 under `DOCKER_ENV_VARS_ALWAYS`.
    104 
    105 #### CFLAGS
    106 
    107 CFLAGS are not automatically passed to docker because they might contain spaces,
    108 '"' or other characters that will require escaping. The solution is to pass it with
    109 `DOCKER_ENVIRONMENT_CMDLINE` and escape every character as required.
    110 
    111 e.g: if wanting to override STRING_WITH_SPACES
    112 
    113 ```
    114 # normal build
    115 CFLAGS=-DSTRING_WITH_SPACES='\"with space\" make
    116 # in docker
    117 DOCKER_ENVIRONMENT_CMDLINE='-e CFLAGS=-DSTRING_WITH_SPACES=\'\\\"with\ space\\\"\'' \
    118  BUILD_IN_DOCKER=1 make
    119 ```
    120 
    121 Alternatively, it is often easier to define the CFLAGS in the Makefile which gets
    122 evaluated inside the Docker image again), conditional on a less complex environment
    123 variable that gets added to `DOCKER_ENV_VARS` in the Makefile.
    124
