    6 Background {#background}
    7 ==========
    8 This page explains how to configure some popular terminal programs for correct
    9 display of newlines when using the serial interface of a RIOT powered device.
    10 When printing something using *stdio* (e.g., `printf("Hello World!\n");`, RIOT
    11 sends a line feed character (`0xA`) as `\n` (newline).
    12 
    13 Some terminals need more, for example, a carriage return and a line feed
    14 character (0xD, 0xA). See https://en.wikipedia.org/wiki/Newline for background.
    15 
    16 This page tries to collect needed settings for common terminal programs that
    17 will make them correctly display newlines.

    19 gtkterm {#gtkterm}
    20 ======
    21 - Graphical method:
    22  - Open the configuration menu.
    23  - Click on ***CR LF auto***.
    24 - Manual method:
    25  - Edit the file `~/.gtktermrc`.
    26  - Change value of ***crlfauto*** option to `True`.
    27 
    28 minicom {#minicom}
    29 =======
    30 - Interactive method:
    31  - Press ***Ctrl+A u***.
    32 - Manual method:
    33  - Edit the configuration file (`~/.minirc.dfl` per default).
    34  - Add the following line:
    35  pu addcarreturn Yes

    37 miniterm {#miniterm}
    38 ========
    39 - Generic method:
    40  - Start with `--eol CR`parameter.
    41 - Via RIOT build system:
    42  - `RIOT_TERMINAL=miniterm make term`
    43 
    44 picocom {#picocom}
    45 =======
    46 - Generic method:
    47  - Start with `--imap lfcrlf` parameter.
    48 - Via RIOT build system:
    49  - `RIOT_TERMINAL=picocom make term`

    51 putty {#putty}
    52 =====
    53 - Graphical method:
    54  - Go to configuration tree and choose `Terminal` branch.
    55  - Enable option `Implicit CR in every LF`.
