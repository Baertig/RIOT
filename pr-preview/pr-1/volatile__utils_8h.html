    Detailed Description

    Utility functions for non-atomic but volatile access.

    Author
    Marian Buschsieweke maria.nosp@m.n.bu.nosp@m.schsi.nosp@m.ewek.nosp@m.e@ovg.nosp@m.u.de

    This functions intentionally have the same signature as the Utility functions for atomic access but don't guarantee atomic access and sequential consistency (or any other consistency). They basically only guarantee that no compiler optimization is applied to that operation and that the compiler will not reorder these operations in regard to each other or other volatile accesses.

    Warning
    In most cases using this over Utility functions for atomic access is a bug!
